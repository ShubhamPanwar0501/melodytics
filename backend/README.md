# Melodytics Backend

The API server for the Melodytics platform, providing song management, data ingestion, and rating features.

## Setup

### Prerequisites
- Python 3.10+
- PostgreSQL

### Local Installation
1. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

2. Environment Configuration:
   Create a `.env` file from the example:
   ```bash
   cp .env.example .env
   ```

3. Run Migrations:
   ```bash
   alembic upgrade head
   ```

4. Start the Server:
   ```bash
   uvicorn app.main:app --reload
   ```

## API Documentation
The API documentation is automatically generated by FastAPI and available at:
- Swagger UI: http://localhost:8000/docs
- ReDoc: http://localhost:8000/redoc

## Key Features
- **Song Search**: Supports case-insensitive partial matching (ilike) via the `/api/songs/search` endpoint.
- **Data Ingestion**: Processes JSON files and upserts data into PostgreSQL.
- **Pagination & Sorting**: Robust pagination and field-based sorting for song listing.
- **Rating System**: Endpoint to update song star ratings with validation.

## Technical Structure
The backend follows a Router-Service-Repository architecture:
- `app/routers`: API endpoint definitions and Pydantic validation.
- `app/services`: High-level business logic and orchestration.
- `app/repositories`: Database interaction using SQLAlchemy ORM.
- `app/models.py`: Database schema definitions.
- `app/schemas.py`: Pydantic models for request/response serialization.

## Database Migrations (Alembic)
Manage schema changes using Alembic:
- Create migration: `alembic revision --autogenerate -m "description"`
- Apply migration: `alembic upgrade head`

## Testing
Run the backend test suite from the project root:
```bash
backend/venv/bin/pytest backend/tests
```

Or, from this `backend` directory with your virtual environment activated:
```bash
pytest tests
```

Generate a terminal coverage report with missing lines:
```bash
pytest tests --cov=app --cov-report=term-missing
```

Generate an HTML coverage report:
```bash
pytest tests --cov=app --cov-report=html
```

Open the HTML report at:
```text
htmlcov/index.html
```
